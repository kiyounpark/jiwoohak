# 지금 우리 학교는 - 생존 유형 테스트

> **Netflix 드라마 지금 우리 학교는 기반으로 만들어진 테스트로 "좀비가 된 학교에서 나는 어떻게 살아남을까?"**  
> 12문항 선택형 테스트 웹앱입니다.  
> 사용자는 간단한 선택지를 통해 캐릭터 성향을 분석받고, 결과 페이지에서 공유 링크를 복사할 수 있습니다.

---

## 이 Repository가 제공하는 기능

- **순수 HTML·CSS·JavaScript**로 동작하는 클라이언트 사이드 테스트 앱
- **12개의 시나리오 기반 질문**과 결과 가중치 계산 로직
- **캐릭터별 결과 페이지 16종** + 요약 결과 페이지 제공
- 진행도 바, 결과 이동 버튼, 링크 복사 등 **UX 요소 포함**
- Netlify 등 **정적 호스팅 환경에서 바로 동작**하도록 구성

---

## 아키텍처 한눈에 보기

```
Client (정적 프론트엔드)
 ├─ HTML: 메인, 질문, 결과 페이지 전부 정적 파일로 구성
 ├─ CSS: 기본/메인/QnA/애니메이션/결과 테마 스타일 시트
 └─ JavaScript: 질문 데이터와 로직(start/js/data.js, start/js/start.js)

Backend
 └─ 없음. 모든 계산과 렌더링이 브라우저에서 수행됩니다.
```

---

## 도메인 기능

- **질문/답변 구성**  
  `start/js/data.js`에서 12문항과 각 선택지의 캐릭터 가중치를 관리합니다.

- **결과 계산**  
  `start/js/start.js`의 `calResult` 함수가 선택지별 가중치를 합산해 상위 캐릭터를 결정합니다.

- **화면 전환 흐름**  
  `begin → goNext → 결과 이동` 흐름으로 질문 페이지와 결과 페이지를 오가며 진행도를 표시합니다.

- **결과 제공**  
  `result-0.html ~ result-15.html`, `resultJun.html` 등  
  캐릭터/요약별 결과 페이지를 **정적 HTML로 제공합니다.**

- **공유 기능**  
  버튼 클릭 시 `clip` 함수로 현재 URL을 복사해 친구와 결과를 공유할 수 있습니다.

## 기술 스택

- HTML5, CSS3
- JavaScript (Vanilla)
- Bootstrap 5 (레이아웃·컴포넌트 활용)
- Netlify 등 정적 사이트 호스팅 호환

---

## 폴더 구조

- `index.html` : 메인 랜딩 페이지
- `start/` : 질문 로직과 리소스 폴더
  - `js/data.js` : 질문·답변 데이터 + 캐릭터 매핑
  - `js/start.js` : 진행 제어, 결과 계산, 공유 링크 복사 로직
  - `img/` : 배경 및 캐릭터 이미지
- `css/` : 공통·메인·QnA·애니메이션·결과 스타일 시트
- `result-*.html`, `resultJun.html` : 캐릭터별 결과 페이지 모음
- `qna.html` : 질문 페이지 템플릿
- `sitemap.xml`, `robots.txt` : 검색 엔진 메타 파일
